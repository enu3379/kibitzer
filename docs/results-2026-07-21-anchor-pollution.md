# Results — OK-anchor 오염 루프 검증과 존폐 결정 (2026-07-21)

원인 문서: [handoff-2026-07-21-anchor-pollution.md](handoff-2026-07-21-anchor-pollution.md).
산출: 실험 결과(본 문서) + **PR #118**(오염 차단 가드) + **PR #120**(anchor
기본 비활성화, 본 문서를 포함한 PR).

모든 실험은 실제 ONNX KoEn-E5-Tiny 임베딩 + 리포 실함수(`tier0_score_parts`,
`anchor_admission_eligible`)를 사용한 합성 세션 시뮬레이션이다. 확률 조건은
40 seeds 평균.

## 1. 오염 루프 재현 (1차)

- **과거 사고는 구형 스택의 산물**: 해시 임베딩 + τ=0.15 조건에서 "킬로그램
  → 무관 7연쇄 false-OK, anchor 단조 이탈"이 완전 재현됨. 현행(ONNX + τ=0.6,
  접미사 스트리핑)에서는 두 수정이 각각 독립적으로 루프를 끊어 재현 불가.
- **현행 스택의 잔존 루프**: 상호 유사한 페이지 클러스터(웹툰 빈지, 상호 cos
  0.75–0.82) + Tier 1 false-OK 1건 편입 → **false-OK 7건 연쇄, 자기강화**
  (cos(anchor,goal) 0.24 고착). "같은 주제로 빠지는" 빈지 패턴이 정확히 점화
  조건.
- 대책 비교: ε 상향은 실패(Tier 1 무조건 편입 분기가 우회), 가중 anchor는
  역효과, **anchor 타이브레이커화가 연쇄를 완전 절단**(7→1, 부작용 0).
- 부수 발견: 전 실험에서 anchor가 관련 페이지를 DRIFT에서 구제한 사례 **0건**.

## 2. 차단 수정 — PR #118 (머지 대상 유지)

`fix: block anchor pollution loop with goal-affinity floors`

1. `anchor_tiebreak_floor`(0.30): 직접 목표 유사도(exemplar/derived)가 하한
   미만이면 anchor 항을 Tier 0 max()에서 제외.
2. `tier1_anchor_floor`(0.30): Tier 1 OK는 전용 게이트만 타며 ε 분기로
   폴스루하지 않음. (최초 구현이 ε 분기에 가려 발동하지 않는 것을 재실험이
   잡아내 분기를 재구성함.)

검증: dev+패치 `pytest` 320 passed, 수정 후 재실험 false-OK 7→1, anchor 청정
유지(cos 최저 0.235→0.814), false-DRIFT·교차언어 recall 부작용 0. **§4의
제거 결정 이후에도 이 가드는 anchor 재활성화 시의 방어선으로 유효하다.**

## 3. anchor 기여도 평가 (2차)

설계 의도(지침서 §4.1)는 "정상 심화(normal deepening) credit" — 목표 문구에
없는 하위 주제를 궤적 근접으로 구제하는 false-drift 완화 장치. 그러나 ONNX
전환의 τ 상향(0.15→0.6)으로 구제 문턱이 `cos(page, anchor) ≥ τ/β = 0.706`이
되어 설계 목적이 구조적으로 불능이 됐다:

- 정상 심화 체인 9단계 시나리오에서 anchor 구제 **0단계** (최근접 0.556).
- anchor는 "exemplar 유사도가 높아 OK된 페이지들의 평균"이라 exemplar와
  상관된 신호 — 구제가 필요한 저유사 관련 페이지와는 애초에 멀다.
- 0.706을 넘는 유일한 실측 사례는 상호 유사도가 극단적인 오염 클러스터(빈지)
  — 즉 τ 상향이 구제력과 오염력을 함께 죽였는데 오염력만 부분 생존해 있었다.
- 원래 담당하던 하위주제·교차언어 구제는 현재 goal enrichment 파생 문구
  (recall 23.8%→86.2%)와 Tier 1 재심이 대체 수행 중.

## 4. 제거 vs 대체 실험 (2차) — 결정: **C1 채택**

| 조건 | 요약 결과 |
|---|---|
| C1 anchor 제거 | 4개 시나리오 중 3개에서 현행과 완전 동률. 유일한 손실 = 교차언어에서 Tier 1 호출 **+1건/세션** |
| C2 Tier 1 OK 승격 (무가드) | **anchor 오염보다 질적으로 악화** — false-OK 1→9, 영속성(롤오프 없음, 후반 재방문도 OK), 감시 우회(오염이 Tier 0 종결 → Tier 1 검토 소멸) |
| C3 승격 + 가드 0.30 | 불충분 — "킬로그램"형(affinity 0.35) 통과. 유효 대역 0.40~0.45는 오염원(0.436)과 여유 0.01 |
| C5 승격 + 안전판 3종 | **C1과 전 시나리오·전 지표 완전 동률** (아래) |

C5(가드 0.40/0.45 + LLM 키워드 검증 오라클 90%/70% + 롤링 윈도 5 +
promoted-only Tier 1 재확인)의 순이득이 0인 것은 우연이 아니라 **구조적**이다:
승격을 안전하게 만드는 재확인 안전판이 승격의 유일한 실익(Tier 1 호출 절감)과
정확히 1:1로 상쇄된다. "안전한 승격"은 "승격 안 함"과 수학적으로 등가이며,
남는 것은 승격 풀·게이트·검증 호출·수명 관리라는 상태와 복잡도뿐이다.

사용자 결정 기준("C5가 C1 대비 순이득이 없으면 C1")이 그대로 발동 →
**anchor를 판정에서 제거(C1)한다.** 구현은 `configs/default.yaml`
`relevance.anchor_window: 10 → 0` (윈도 0이면 anchor 미형성 → 판정에서 소멸;
코드는 유지되어 replay 비교와 롤백이 자유롭다).

## 5. 남은 것

- **실로그 확인**: 도그푸딩 로그가 쌓이면 replay CLI로 (a) anchor 제거로
  늘어난 Tier 1 호출량이 허용 범위인지, (b) anchor 구제율이 실로그에서도
  0인지 확인. 0이 아니면 `anchor_window` 복원 + PR #118 가드 체제로 회귀.
- 승격안을 재검토할 경우의 전제(실험으로 확인): 재확인을 일반 Tier 1보다 싼
  경량 호출로 설계하거나, 컨트롤러 r_final·개입 타이밍에 주는 2차 효과까지
  모델링해 손익이 달라지는지 봐야 한다.
- 실험 스크립트: 세션 스크래치패드 `sim_anchor_pollution.py`,
  `sim2_regimes.py`, `sim3_binge.py`, `sim4_fix_verify.py`,
  `sim5_promotion.py`, `sim6_c5.py` (세션 종료 시 소멸 — 재현 필요 시 본
  문서의 시나리오 정의로 재작성).
