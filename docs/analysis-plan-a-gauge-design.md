# A안 대체 설계 — 몰입 게이지(S) 컨트롤러

작성: 2026-07-21. 코드 기준: `agent/windows-toast-notifications` 브랜치 (b7dc8c7).

## 지위 (Status)

- **이 설계는 A안(`AlignmentController`, EWMA + 히스테리시스)을 완전히 대체한다.**
  `docs/analysis-plan-a-redesign.md` §3(재설계안)~§5(미해결 질문)를 supersede하며,
  같은 문서의 §1~§2(현황 분석·문제점 P1~P10)는 이 설계의 근거 기록으로 유효하다.
- **캘리브레이션(D4 Replay CLI) 완료 후 출하 기본 컨트롤러가 된다** — 현행 기본
  `type: "streak"`(B안)을 대체하는 것이 최종 목표다.
- B안과의 호환성은 설계 조건이 아니다. B안은 이 설계 안정화 후 제거 또는
  실험용으로 강등될 예정이며, 공유 컴포넌트 유지를 위해 이 설계를 타협하지 않는다.
- 병행 별도 과제: TIER 0/1 오판정 개선(별도 작업으로 진행 — 게이지는 오판을
  관성으로 완충할 뿐 판정 품질을 대신 고치지 않는다), 누적 드리프트 시간
  조건. 시간 조건 과제의 "무감쇠 예산 클록" 전제는 **이 설계와 충돌하므로
  이 문서가 우선한다**(§9 참조). 칭찬(celebration) 발동 조건은 별도 과제에서
  이관되어 **본 설계 §6이 정의한다**.

## 1. 설계 개요

세션마다 **몰입 게이지 S ∈ [0, 100]** 하나를 둔다. 기본값 100.

- 이탈(DRIFT) 페이지를 읽는 동안 S가 체류 시간에 비례해 감소한다.
- 이탈이 **연속될수록**(시간의 연속 또는 연속된 이탈 페이지) 감소가 가팔라진다
  — 관성(momentum)에 의한 이산 가속 단계.
- 한 번의 복귀로는 가속이 풀리지 않고, **연속된 복귀**에만 회복 가속이 붙는다.
- **S = 0 도달이 나깅 후보 시점**이다. 최종 확정은 Tier 2 Judge가 한다.
- 페이지 단위 판정은 이진(OK/DRIFT, Tier 0/1 기준)으로 소비하고, 훈수 여부는
  그 판정 하의 **체류 시간**이 결정한다.

이 단일 스칼라가 기존 A안의 EWMA(자격)와 D7 누적 클록(타이밍)을 모두 흡수한다.
"소모된 예산은 회복하지 않는다"는 회계적 관점은 폐기한다 — S는 복귀로 회복되는
**현재 몰입 상태**이며, 흩어진 이탈은 사이사이의 회복 때문에 0에 도달하지 못한다.
이것이 P1(정렬 양호 세션에 대한 오탐 나깅)의 구조적 해소다. 연속성 가중은 B안
streak의 직관을 연속량으로 흡수한 것이다.

### 기존 문제(P1~P10) 해소 매핑

| 문제 | 해소 방식 |
|---|---|
| P1 EWMA 트리거 이탈 | 게이지 자체가 유일한 트리거. "정렬도 낮을 때만 개입"이 S=0 조건으로 복원 |
| P2 체류 시간 무감각 | S는 heartbeat 시간 적분으로 감소. 40분 응시 1페이지 ≫ 3초 스침 10페이지 |
| P3 r 스케일 혼합 | r_final 매핑층 폐기, 이진 verdict만 소비 (§3). 축퇴 모드만 마진 가중 |
| P4 근거 없는 상수 | 모든 노브를 placeholder로 명시(§8)하고 D4 캘리브레이션 전까지 확정하지 않음 |
| P5 회복 의미론 이중화 | 상태가 S 하나뿐 — 회복 규칙도 하나(§4). 무감쇠 클록 폐기 |
| P6 streak 컬럼 재사용 | 신규 `gauge_states` 테이블 — 기존 스키마와 얽힘 없음 (§2) |
| P7 수식 복제 | 갱신 로직은 `GaugeController` 단일 소유. 칭찬 판정도 컨트롤러 API 경유 |
| P8 관측 횟수 콜드스타트 | 콜드스타트 게이트 폐지 — S=100 자체가 완충. 단일 페이지 세션도 도달 가능 |
| P9 경로별 재나깅 상이 | 트리거 경로 단일화(§6). 레거시 이벤트 경로는 gauge 타입에 없음 |
| P10 검증 부재 | 섀도 운전 단계(§9 1단계)로 실사용 데이터 확보 후 기본 전환 |

## 2. 상태 모델

```text
GaugeState  (신규 테이블 gauge_states — controller_states와 분리)
  s                : float    # 몰입 게이지 ∈ [0, 100], 초기 100
  m                : float    # 관성 ∈ [-1, +1], 초기 0. +1 = 이탈 지속, -1 = 복귀 지속
  accel_tier       : int      # 이산 가속 단계 ∈ {0, 1, 2}, 초기 0
  updated_at       : datetime # 마지막 적분 시각
  last_nag_ts      : datetime | None
  nag_n            : int      # 이번 에피소드의 나깅 순번 (m ≤ 0에서 리셋)
  renag_debt       : float    # 마지막 나깅 이후 이탈 부채 적산 — 재나깅 스케줄용 (§6)
  celebrate_armed  : bool     # 칭찬 대기 — S ≤ C_arm에서 set, 발송 시 clear (§6)
  snoozed_until    : datetime | None      # 사용자 명시 스누즈 (유일한 외부 게이트)
  last_judgment    : (page_key, verdict, ts) | None   # Tier 2 판정 캐시 (§5)
```

기존 A안 상태(`alignment_score`, `drift_latched`, `armed`, `obs_count`,
`coldstart_observations`)와 D7 모드 클록(`cumulative_drift_seconds`)은 gauge
타입에서 사용하지 않는다. cooldown은 재나깅 부채 스케줄(§6)이 대체하므로 별도
상태가 없다.

## 3. 입력 신호 — r_final 폐기, 이진 verdict

**r_final 매핑층(`RELATED_RELEVANCE=0.85`, `TIER1_DRIFT_RELEVANCE=0.0`,
`relevance.py:9-20`)은 폐기한다.** 이 상수들은 구 EWMA를 θ_low 아래로 보내기 위해
역산된 접착제였고(0.0의 코드 주석이 스스로 증언), 새 모델에는 존재 이유가 없다.
원시 코사인 r0는 지금처럼 features 진단·리플레이 기록으로만 보존한다.

### 3.1 정상 모드 (Tier 1 가동)

컨트롤러는 관측의 최종 verdict만 소비한다:

| 판정 | 방향 | 속도 배율 |
|---|---|---|
| OK (Tier 0 통과 / Tier 1 구제 / D8 라벨 related) | 회복 | 1.0 (전속) |
| DRIFT (Tier 1 확정 / D8 라벨 drift) | 감소 | 1.0 (전속) |

Tier 0 OK는 Tier 1 검토를 거치지 않지만(현 구조상 Tier 1은 Tier 0 DRIFT의 구제
심사만 함) 전속 회복으로 믿는다. **비대칭의 근거**: 가짜 OK의 비용은 "나깅
지연"(덜 나쁜 실패)이고 가짜 DRIFT의 비용은 "잘못된 나깅"(최악의 실패)이다 —
오탐 최우선 원칙(지침서 §1-1)에 부합한다. 단발 오판은 관성(§4)이 완충한다.

### 3.2 축퇴 모드 (LLM 미가용 — Tier 1/2 다운)

Tier 0 코사인은 캘리브레이션된 신뢰도가 아니므로(tau_ok부터 미검증 잠정값),
축퇴 모드에서는 **임계값 마진의 볼록 함수**를 속도 배율로 곱해 보수적으로 운전한다:

```text
margin = |r0 - tau_ok|
f(margin) = clamp((margin / M)^p, 0, 1)      # placeholder: p = 3, M = 0.25
# 예: margin 0.05 → f ≈ 0.008 (거의 무시), margin 0.20 → f ≈ 0.51
```

애매한 판정(임계 근처)은 게이지를 거의 움직이지 못하고, 확실한 판정만 반영된다.
임계 근소 미달 페이지를 확정 이탈과 동일하게 취급하던 문제의 축퇴 모드 해법이기도 하다.
마진 스케일은 임베딩 모델 의존이므로 모델 교체 시 M 재보정이 필요하다(§8).

이 모드에서는 Tier 2도 없으므로 가속 승격 검증(§5)과 S=0 최종 관문 없이
게이지 단독으로 나깅한다(마진 가중이 유일한 안전판 — 그래서 볼록이어야 한다).
축퇴 모드의 진입·해제는 기존 provider degraded 신호(Tier 1 연속 실패/타임아웃
감지 — `docs/planning-notes.md` Evidence 절의 `provider.degraded`)를 재사용한다.

## 4. 동역학

heartbeat(presence)와 관측 이벤트마다 동일한 `advance(now)`를 호출한다:

```text
advance(now):
  Δ = clamp(now - updated_at, 0, gap_cap)          # 자리 비움/타 앱 사용 시 정지
  d = 현재 활성 페이지 verdict가 DRIFT ? +1 : -1
      # 페이지 전환은 즉발 효과가 없다 — 활성 verdict의 교체 시점은 새 관측의
      # 판정 확정 시점이며, 디바운스·판정 지연 동안은 직전 verdict로 적분한다.
  w = 축퇴 모드 ? f(margin) : 1.0

  # 1) 관성 적분 — 방향 전환은 느리다 (한 번의 복귀/이탈로 안 풀림)
  m ← m + (d - m) · (1 - exp(-Δ / tau_m)) · w

  # 2) 가속 단계 전이 — 히스테리시스, 승격은 Tier 2 검증 필요 (§5)
  m ≥ T_up[tier]  → 승격 후보 (정상 모드: Tier 2 confirm 시 승격 / 축퇴 모드: 즉시 승격)
      # Tier 2 응답 대기 중(최대 180s)에는 현재 단계 배율로 적분을 계속하고,
      # confirm 도착 시점부터 새 배율을 적용한다(소급 재계산 없음). 대기 중 중복 승격 후보는 무시.
  m ≤ T_down[tier] → 즉시 강등

  # 3) 게이지 적분
  DRIFT 중:  s ← max(0,   s - R_drain · A[accel_tier] · w · Δ)
  OK 중:     s ← min(100, s + R_recover · ((1 - m) / 2) · w · Δ)

  updated_at ← now
```

설계 성질:

- **연속성 가중**: 이탈이 지속되면 m이 +1로 올라가 단계 승격 → 감소 가속.
  같은 총 이탈 시간이라도 연속 이탈이 흩어진 이탈보다 먼저 0에 닿는다.
- **회복 관성 (연속형, 2026-07-21 채택)**: 회복 속도가 `(1 - m) / 2`에
  비례한다 — m = +1(이탈 관성 만점)이면 0, m = 0이면 50%, m = -1이면 100%.
  복귀 직후부터 회복이 미세하게 시작되어 관성이 풀릴수록 가속되므로,
  "시간을 내면 오른다"는 체감 인과가 화면에서 끊기지 않는다. 그래도 복귀
  1분의 회복은 1점 미만이라 "한 번의 복귀로 가속이 안 풀린다"는 성질은
  유지되고, 단발 Tier 1 오판(가짜 OK 1건)이 누적 증거를 지우던 문제도
  같은 메커니즘으로 완충된다. (초기안이던 하드 컷 `max(-m, 0)`은 복귀 첫
  3~4분간 게이지가 완전 동결되는 체감 인과 위반 때문에 기각.)
- **콜드스타트 불필요**: 세션 시작 시 S=100이라 어떤 경로로도 즉시 나깅은
  불가능하고, 목표 선언 직후 이탈 페이지 하나에 계속 머무는 세션도 dwell
  적분으로 결국 0에 도달한다(관측 횟수 게이트 폐지 — P8 해소, finding 1
  시나리오 성립).
- **전 페이지 dwell 추적 전제**: 현재 presence 파이프라인은 OK 페이지
  heartbeat를 조기 반환한다(`observations.py:421-428`). S 회복과 페이지별
  dwell 기록(§5.3)을 위해 OK 페이지 heartbeat도 게이지 갱신까지 통과시킨다.

## 5. Tier 2 — 질문 재정의와 이중 관문

### 5.1 질문 재정의

Tier 2 Judge의 질문을 "지금 개입할까?"(notify/defer)에서 **"이 흐름이 실제
이탈인가?"**로 바꾼다. 판정 스키마:

```text
{"flow": "drift" | "ok", "reason_code": "off_goal | useful_side_branch | insufficient_evidence", "basis": "title | content | both"}
```

`insufficient_evidence`는 `flow: "ok"` 취급(오탐 최우선). 나깅 문구 Writer는
현행대로 분리 유지 — Judge가 drift 확정한 뒤에만 호출된다.

### 5.2 호출 시점 — 두 곳, 호출 수 유계

**(a) 가속 단계 승격 시 (검증 관문).** m이 T_up을 넘어 승격 후보가 되는 순간
호출한다. 승격은 이산 사건이므로 호출 수는 에피소드당 최대 단계 수로 유계다.

- `flow: drift` → 승격 확정.
- `flow: ok` → 승격 취소 + 관성 감속(`m ← min(m, T_down[tier])`) + 소폭 환급
  (`s ← s + B_refund`) + **유효 verdict 오버라이드**(아래). Tier 1 연속
  오판에 의한 오가속을 조기에 끊는 브레이크.
- 판정은 결과와 무관하게 `last_judgment`에 캐시한다.

**유효 verdict 오버라이드**: Tier 2는 티어 서열상 최상위 증거(발췌문·타임라인·
dwell)로 판단하므로, `flow: ok`는 하위 판정을 그냥 뒤집는다 — Tier 1이
Tier 0의 DRIFT를 구제하는 기존 패턴의 상향 연장이며, **시한 없이 해당
페이지의 활성 체류가 끝날 때까지(페이지 전환까지)** 유효 verdict가 OK가 된다.
이후 그 페이지는 여느 OK 페이지와 동일하게 취급된다(적분 방향 회복, 게이지
침묵) — Tier 0 OK 페이지에 무한 체류해도 침묵하는 기존 규칙과 같으므로
예외가 아니다. 오버라이드는 게이지 계층에서만 작동한다: 관측 레코드
(verdict·r0)는 불변으로 보존되고(감사·리플레이), anchor·exemplar에도 들어가지
않으며(앵커 오염 루프 격리), 소급 재계산은 여전히 D8 라벨의 몫이다. 페이지를
떠났다가 재방문하면 새 관측이 정상 파이프라인(Tier 0/1, 필요시 Tier 2)을
다시 탄다. Tier 2 자신의 가짜 ok는 해당 체류 동안 지속되는 비용이 있으나,
"가짜 OK의 비용 = 나깅 지연"이라는 오탐 최우선 원칙(§3.1)의 수용 범위다 —
교정 경로는 페이지 전환과 D8 라벨.

부수 효과: 승격 시점에 excerpt를 수집하므로 S=0 시점의 판정 지연(Tier 2
타임아웃 최대 180s)이 자연 해소된다(프리페치).

**(b) S = 0 도달 시 (최종 관문).** 나깅 후보. 단, **현재 페이지에 대한 신선한
판정 기록이 있으면 재판정을 생략**한다:

```text
S = 0 도달:
  last_judgment.page == 현재 페이지 AND (now - ts) ≤ fresh_window
    → 캐시된 flow 판정 재사용 (재호출 없음)
  아니면 → Tier 2 호출
  flow drift → Writer 호출 → 나깅 발송 (S는 0 유지 — 재나깅은 §6 부채 스케줄)
  flow ok    → s ← R_dismiss (오판 교정 환급), m ← min(m, 0), accel_tier ← 0,
               유효 verdict OK 오버라이드 ((a)와 동일 — 이후 S는 회복 방향)
```

최악 케이스 호출 수: 실제 이탈 에피소드당 (가속 단계 수 + 1)회. 오판(ok)
페이지는 오버라이드 이후 그 체류 동안 추가 호출이 없다. fresh_window 캐시는
같은 페이지에 대한 중복 호출 억제(S=0 관문·재방문 직후)에만 쓰인다.

### 5.3 페이로드 — 현행 + 페이지별 체류 시간

현행 페이로드(`tier2_payload.py:27-68` — goal, 현재 페이지 title/host/verdict/
excerpt(≤3000자), 최근 30개 제목 타임라인, 최근 페이지 excerpt, 재방문 신호)를
유지하고 **페이지별 `dwell_seconds` 하나만 추가한다**:

```text
current:  title, url_host, verdict, dwell_seconds, page_excerpt
timeline: [{title, verdict, dwell_seconds, repeat_count}]   # 압축 규칙 현행 유지
gauge:    {s, accel_tier, episode_drift_seconds}            # time_budget 컨텍스트 대체
```

dwell이 "흐름" 판정의 핵심 증거다 — 3초씩 스친 곁가지와 20분 응시한 이탈을
제목 목록만으로는 구분할 수 없다. 문서 전문 전송은 하지 않는다(비용·지연 대비
한계효용 낮음, excerpt 현행 유지).

## 6. 나깅·리필·재나깅·칭찬

- **나깅은 S를 건드리지 않는다** (design call, 2026-07-21): S는 집중도
  게이지이므로 시스템이 임의로 올리지 않는다. 이탈이 지속되면 S는 0에
  클램프된 채 유지된다 — 팝업 표시가 현실을 반영한다. 재나깅은 별도의
  **이탈 부채 카운터**로 스케줄한다: 나깅 시 `renag_debt ← 0`, 이후 S 감소와
  동일한 적분(`R_drain · A[tier] · w · Δ`)을 부채로 쌓아
  `renag_debt ≥ min(R_renag · B_backoff^(n-1), R_renag_max)`이면 다음 나깅
  후보. 동역학은 리필 방식과 수학적으로 동일하되 S의 의미만 보존한다.
  부채는 게이지 상한(100)에 묶이지 않으므로 백오프 사다리가 2단 가속 기준
  **약 2.7분 → 5.3분 → 10.7분 → 21.3분(상한)**으로 자연 확장된다. 에피소드
  종료(m ≤ 0)에 순번 n과 부채를 리셋한다. 무시당한 훈수일수록 다음 훈수를
  아끼는 것이 훈수 피로 방지와 오탐 최우선 원칙(지침서 §1-1)에 부합한다.
- S=0에서 Judge가 ok 판정 → `s ← R_dismiss` (placeholder 30) + 관성·단계 리셋.
  이것은 임의 리필이 아니라 **오판 교정**이다 — Judge가 이탈이 아니라고 본
  만큼 잘못 깎인 집중도를 되돌린다(승격 기각의 `B_refund`도 동일한 성격).
  "기각됐는데 S가 0에 머물러 즉시 재후보"가 되는 루프도 차단한다.
- 스누즈(사용자 명시)는 유일한 외부 게이트로 유지 — 스누즈 중에는 나깅·Tier 2
  호출만 억제하고 게이지 적분은 계속한다(스누즈 해제 시 상태가 현실을 반영하도록).
- **훈수 상한은 두지 않는다** (design call, 2026-07-21): 백오프 사다리가
  부채 상한(R_renag_max)에 도달한 뒤에는 같은 주기(2단 가속 기준 약 21분)로
  계속 발송한다. 그 이상의 침묵은 시스템이 아니라 **사용자의 스누즈 호출에
  위임**한다 — 알람이 스스로 꺼지는 감시 도구는 신뢰가 깨지고, 지겨움의
  해소는 사용자 주권(스누즈)의 몫이다.
- **칭찬(celebration)** (2026-07-21 개정): **S가 `C_arm`(placeholder 20)
  이하로 떨어졌던 에피소드에서, S가 `C_celebrate`(placeholder 80) 이상으로
  처음 회복**되면 1회 복귀 칭찬을 발송한다. 대기 플래그는 S ≤ C_arm에서 set,
  발송 시 clear — 재발동은 다시 C_arm 이하로 떨어져야 가능하다(20↔80의 넓은
  히스테리시스라 경계 플랩 없음). **나깅 발생은 조건이 아니다**: 나깅 전에
  스스로 복귀한 회복(가장 칭찬받을 행동)을 포착하며, 나깅을 받은 에피소드의
  회복은 자동 포함된다(나깅은 S=0 ≤ C_arm 경유). 칭찬의 희소성은 나깅 여부
  대신 C_arm의 깊이(게이지 80% 소진)가 통제한다. 나깅이 S를 건드리지 않으므로
  S ≥ C_celebrate는 회복 적분으로만 도달 가능하고, 그 자체가 십수 분의 지속
  복귀를 요구하므로 별도의 에피소드 종료 게이트가 필요 없다. C_arm →
  C_celebrate(60점) 회복은 placeholder 기준 지속 복귀 약 15~18분(§8 시간 앵커).

## 7. B안·기존 경로 처리

- `controller.type: "gauge"`를 신설한다. gauge 타입은 자체 트리거 루프(§4~§6)만
  가지며 레거시 이벤트 경로(`REQUEST_EXCERPT` armed 트리거)와 D7 시간 경로
  어느 쪽에도 얹히지 않는다 — P9(경로별 재나깅 상이)가 원천 소멸.
- **D7 시간 예산 규칙은 gauge 타입에서 흡수·대체된다**: 모드 클록
  (`cumulative/continuous_drift_seconds`)·리뷰 경계(`next_review_mode_seconds`)·
  per_page/total 임계는 게이지 S와 재나깅 부채가 대신한다. 재사용하는 것은
  presence 파이프라인(heartbeat·gap cap)과 "목표 시간 → 예산" 개념(§8 —
  단 선형 1/6에서 준선형으로 변형)뿐이다. streak 모드가 남아 있는 동안
  D7 경로는 streak 전용으로 유지된다.
- `type: "alignment"`는 gauge 안정화 즉시 제거한다(마이그레이션: alignment 설정
  세션은 gauge로 승격). `type: "streak"`(B안)은 캘리브레이션 완료·기본 전환
  시점까지 유지 후 제거 또는 실험용 강등 — 별도 결정(§10).
- D8 페이지 라벨 수정: verdict가 소급 변경되므로 관측 타임라인 + 저장된
  페이지별 dwell로 `advance`를 재생(replay)해 S·m을 재계산한다. presence
  이벤트는 타임라인에 없으므로 관측 간 간격을 Δ 근사로 쓴다(gap_cap 적용).
- 칭찬(celebration): `ingest.py`의 복제 수식(`_next_alignment_score`,
  `_drift_confirmed_after_observation`)을 제거하고 §6의 게이지 조건
  (`C_arm` 하락 후 `C_celebrate` 회복)으로 교체한다. 기존 칭찬 발동 조건
  별도 과제 분석의 트리거 부분은 본 설계가 대체한다.

## 8. 노브 목록 (전부 placeholder — D4 캘리브레이션 전 확정 금지)

| 노브 | placeholder | 의미 |
|---|---|---|

| `A[tier]` | [1.0, 1.5, 2.5] | 가속 단계별 감소 배율 — 단계 수는 배열 길이(T_up/T_down과 함께)가 결정 |
| `R_recover` | 100/T_budget (≈0.1 /s) | 회복 만점(m=-1) 기준 — T_budget 연동, 절대 노브 아님 (시간 앵커 참조) |

| `B_ref`/`T_ref`/`B_max` | 15분 / 2h / 30분 | 준선형 앵커점과 예산 상한·하한(min_total 5분) — 시간 앵커 참조 |
| `tau_m` | 300 s | 관성 시간상수 — 방향 전환 체감 약 3~4분 |
| `T_up` | [0.5, 0.8] | 단계 승격 관성 임계 (0→1, 1→2) |
| `T_down` | [0.2, 0.5] | 단계 강등 관성 임계 (히스테리시스) |
| `gap_cap` | presence 클록과 동일 | heartbeat 공백 상한 — 초과분 무시 |
| `R_renag` | 40 | 첫 재나깅 부채 기준 (2단 가속 기준 약 2.7분) |
| `B_backoff` | 2.0 | 재나깅 부채 배수 (40 → 80 → 160 → 320) |
| `R_renag_max` | 320 | 부채 상한 — 재나깅 간격 상한 약 21분 (2단 가속 기준) |
| `C_arm` | 20 | 칭찬 대기 arm 임계 — 이 이하로 떨어졌던 에피소드만 칭찬 대상 |
| `C_celebrate` | 80 | 복귀 칭찬 발동 회복 임계 — C_arm→C_celebrate 회복 약 15~18분 |
| `R_dismiss` | 30 | S=0 기각 시 환급 |
| `B_refund` | 10 | 승격 기각 시 환급 |
| `fresh_window` | 600 s | Tier 2 판정 캐시 신선도 |
| `p`, `M` | 3, 0.25 | 축퇴 모드 마진 가중 (M은 임베딩 모델 교체 시 재보정) |

**시간 앵커 — 세션 목표 시간 준선형 연동**: 세션에는 선언된 목표 시간이
존재한다(`available_time_minutes` — D7은 선형 `total = 목표 시간 × 1/6`을
썼다, `time_budget.py:19-36`). 게이지는 선형 대신 **준선형(sublinear)**으로
연동한다 — 세션이 2배 길다고 이탈 허용량이 2배가 되는 것은 과하다(주의
리듬과 훈수 1회의 비용은 세션 길이와 무관하다):

```text
T_budget = clamp(B_ref · (목표 시간 / T_ref) ^ budget_alpha, min_total, B_max)
  placeholder: T_ref = 2h, B_ref = 15분, budget_alpha = 0.585
               ("세션 2배 → 예산 1.5배"), min_total = 5분, B_max = 30분
  미선언 폴백: 15분 (2시간 세션 상당)
예: 30분 세션 → 6.7분, 1시간 → 10분, 2시간 → 15분, 4시간 → 22.5분
    (D7 선형 1/6이면 4시간에 40분 — 과함)

R_drain   = 100 / T_budget     # 1× 연속 이탈 시 T_budget 만에 100 → 0
R_recover = 100 / T_budget     # 회복 만점(m=-1) 기준 동일 스케일
```

S 단위 노브(재나깅 부채 사다리 `R_renag`·`B_backoff`, `C_celebrate`,
`R_dismiss`, `B_refund`)는 그대로 두어도 시간 표현이 자동으로 비례
스케일된다(부채 40 = T_budget의 40% ÷ 가속 배율). 반면 **사람의 주의 리듬에
속하는 노브는 절대 시간을 유지**한다: `tau_m`, `T_up`/`T_down`, `gap_cap`,
`fresh_window` — 세션이 길다고 "진짜 복귀" 판정이 느려질 이유는 없다.

체감 예시(연속 이탈 첫 훈수 / 칭찬 지속 복귀): 1시간 → T_budget 10분 →
약 8분 / 10.5분. 2시간 → 15분 → 약 10분 / 14분. 4시간 → 22.5분 →
약 13분 / 18분. min_total(5분)이 짧은 세션의 과민화를 막는 바닥이고,
B_max(30분)가 초장기 세션의 방임을 막는 천장이다. 위 표의
`R_drain`/`R_recover` 0.1은 T_budget ≈ 16.7분 상당의 대표값이다.

지침서 §8("튜닝을 라이브로 하지 않는다") 준수: 위 값으로 섀도 운전·도그푸딩은
하되, 확정은 D4 Replay CLI의 라벨된 코퍼스 캘리브레이션으로만 한다.

   시점 결정.
5. **칭찬 임계의 거리** — 연속형 회복 채택 + C_arm/C_celebrate = 20/80 방식
   (2026-07-21 개정 — 나깅 종속 폐지, 자발 복귀 포착)으로 칭찬까지 약 15~18분
   지속 복귀. 남는 것은 실측 검증: 실제 세션에서 칭찬 빈도가
   적정한지(너무 희귀하거나 남발되지 않는지) 도그푸딩·D4로 확인. T_budget
   연동 스케일(§8 시간 앵커)의 적정성 — 특히 짧은 세션에서 min_total(5분)
   바닥에 걸릴 때의 체감 — 도 같은 데이터로 판단. 회복식 분모(현행 2, 즉
   `(1-m)/k`의 k)는 시뮬레이터에서 인터랙티브 검토 중 — k=2 외 값으로
   확정되면 정식 노브로 승격한다.
6. **목표별 tau_ok 비이식성** — 짧고 넓은 목표는 관련 페이지의 유사도 분포
   자체가 낮게 깔려, 축퇴 모드 마진의 절대 스케일이 목표 특성에 따라 달라지는
   문제. 본 설계 범위 밖(Tier 0/1 과제)이나 M 캘리브레이션과 접점 있음.
7. **페이지 전환 임펄스 (확장 포인트, 기본 비활성)** — 관측 확정 시
   `m ← clamp(m + J_page · d)` 임펄스를 추가하면 "여러 이탈 페이지를 옮겨
   다니는" 의도적 서핑 신호를 보강할 수 있다(`J_page=0`이 현행과 동일 —
   행동 불변으로 노브만 선반영 가능, D8 리플레이 호환). 단 관측 횟수
   민감성(P2)의 부분 재도입이므로, 도입 시 에피소드당 임펄스 총량 상한 또는
   호스트 전환 한정 조건이 필요. 섀도 운전에서 페이지 호핑형 이탈의 나깅
   지연이 관측되면 검토.
